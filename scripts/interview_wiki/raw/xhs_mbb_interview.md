# 什么是MBB面试？

MBB = 麦肯锡 + BCG + 贝恩

面试通常分两个部分：

**Case Interview**（案例面试）
→ 给你一个商业问题，现场分析

**Behavioral / PEI**（行为面试）
→ 用真实经历证明你的能力

两个都是一票否决制，缺一不可

---

# Case Interview 三大题型

**① 利润分析 Profitability**
利润 = 收入 - 成本
→ 收入端：价格 × 销量，按产品线拆
→ 成本端：固定 vs 可变，按环节拆

**② 市场进入 Market Entry**
→ 外部：市场规模、增速、竞争格局
→ 内部：能力、资源、战略匹配度

**③ 并购 M&A**
→ 财务：估值倍数、协同效应
→ 战略：业务互补性
→ 风险：整合难度、政策风险

# Case的核心不是背框架

很多人一上来就背框架
面试官一眼就看出来了

真正考的是 **拆问题的能力**：

1. 花30秒想清楚核心矛盾
2. 搭自己的分析结构（不是套模板）
3. 提出假设，用数据验证
4. 敢问 clarifying question
5. 算完要 sanity check

记住：**框架是工具，不是答案**

---

# 行为面试 STAR 法则

**S** - Situation 背景是什么
**T** - Task 你的任务/角色
**A** - Action 你具体做了什么
**R** - Result 结果和数据

提前准备 3-5 个故事，覆盖：

✓ 领导力（带团队解决问题）
✓ 冲突处理（和队友/客户意见不合）
✓ 失败经历（踩坑后怎么爬起来）
✓ 影响力（说服别人接受你的方案）

每个故事控制在 2 分钟内讲完

---

# 准备时间线

**面试前 3 个月**
→ 学框架，熟悉题型
→ 每天做 1 个 case
→ 练 mental math（每天10分钟）

**面试前 1 个月**
→ Mock interview 每周 3-4 次
→ 找同水平的 partner 互练
→ 开始准备行为面试故事

**面试前 1 周**
→ 回顾错题本
→ 不刷新题，巩固旧题
→ 调整作息和心态

---

# 最容易踩的 5 个坑

**❶ 算数太慢**
→ 面试里没计算器，提前练 mental math

**❷ 只说框架不说 insight**
→ 分析完要给结论，不是列完框架就结束

**❸ 行为面试讲太长**
→ 控制在 2 分钟，先讲结果再展开

**❹ 不敢问问题**
→ 问 clarifying question 是加分项

**❺ 只练 case 不练 fit**
→ PEI 是一票否决，别抱侥幸心理
